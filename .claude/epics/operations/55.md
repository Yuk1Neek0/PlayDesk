# Task: Handoff queue UI — /admin/handoff

## Description

New staff-only page rendering the handoff queue with `Take over` and `Mark resolved` actions. Reuses the existing `pd-*` styles. Auto-polls (same 12s cadence as `/admin`).

## Acceptance Criteria

- [ ] `/admin/handoff` renders the queue, newest-first.
- [ ] Each row shows: id, customer identifier, latest message preview (1 line), handoff reason chip, age (relative time), action buttons.
- [ ] `Take over` button calls the take-over endpoint and removes the row from the live polling result on success.
- [ ] `Mark resolved` opens a small modal accepting an optional one-line note, then calls resolve.
- [ ] Non-staff visitors are redirected to `/login` (matches existing /admin gate).

## Technical Details

- `frontend/src/app/admin/handoff/page.tsx`.
- New `frontend/src/components/handoff-row.tsx`.
