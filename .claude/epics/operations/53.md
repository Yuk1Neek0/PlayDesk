# Task: Handoff triggers + early-return guard

## Description

In `backend/agent/loop.py`, detect the conditions defined in the PRD (two consecutive tool failures, iteration cap, human-request keywords, `unsafe_action` structured tool error) and flip `Conversation.status` to `needs_human` with a populated `handoff_reason`. Separately, early-return from `AgentLoop.run()` when the conversation's status is not `active` — so a customer message into a handed-off conversation simply gets stored but does not trigger an LLM call.

## Acceptance Criteria

- [ ] Two consecutive tool errors flip status to `needs_human` with `handoff_reason="repeated_tool_failure"`.
- [ ] The 6-iteration cap (existing) now also flips status with `handoff_reason="iteration_limit"`.
- [ ] User messages matching `/(speak\s+to|human|manager|refund|complaint|说人|找经理|投诉)/i` flip status with `handoff_reason="user_request"`.
- [ ] A tool returning a structured `unsafe_action` error flips status with `handoff_reason="unsafe_action"`.
- [ ] `AgentLoop.run()` early-returns when `conversation.status != 'active'` and writes a `system` Message noting the skip.
- [ ] Five trigger tests + one early-return test, all green.

## Technical Details

- Trigger detection lives in the outer harness of `AgentLoop.run`, not inside any tool — so a misbehaving LLM cannot suppress it.
