# Task: Active-prompt resolver — get_active_config + AgentLoop wiring

## Description

New module `backend/agent/prompt_config.py` exposing `get_active_config(name='default')` that returns the row with `is_active=True` (or falls back to a `LiteralPromptConfig` built from `SYSTEM_PROMPT` when no row exists). `AgentLoop._build_system_prompt` reads it once per turn and stamps `prompt_version` onto every Message it writes. Switching `is_active` takes effect on the next turn.

## Acceptance Criteria

- [ ] `get_active_config('default')` returns the seeded row after the migration.
- [ ] With no `PromptConfig` rows, returns a `LiteralPromptConfig` instance carrying `SYSTEM_PROMPT`, `model_name=settings.ANTHROPIC_MODEL`, `version='default'`. The existing test suite passes through this path.
- [ ] After toggling `is_active`, the next `AgentLoop.run()` writes Messages whose `prompt_version` reflects the new version.
- [ ] At least two new unit tests: fallback path + switch behaviour.

## Technical Details

- Lightweight cache scope: lookup once per `AgentLoop.run()` invocation; do not module-cache (so multi-store / future per-name lookups stay simple).
- Conflicts with task 53 because both edit `agent/loop.py`. Run after 53 lands, OR coordinate by branch order — `conflicts_with: [53]` is set explicitly.
