# Task: Handoff queue API — list / take-over / mark-resolved

## Description

Three staff-only endpoints supporting the handoff queue UI:
- `GET /api/admin/handoff/` — conversations with `status in (needs_human, staff_takeover)`, newest first
- `POST /api/admin/handoff/{id}/take-over/` — flip to `staff_takeover`
- `POST /api/admin/handoff/{id}/resolve/` — flip to `closed`, store the optional resolution note

## Acceptance Criteria

- [ ] List endpoint returns the correct subset, newest first.
- [ ] `take-over` is idempotent; calling twice keeps status `staff_takeover`.
- [ ] `resolve` accepts `{ note?: string }` and writes to `Conversation.resolution_note` (new nullable field — add as a one-line migration alongside 51 if not already there).
- [ ] All three endpoints reject non-staff with 403.

## Technical Details

- `backend/api/admin_handoff.py`.
- Reuse the existing staff-permission class.
