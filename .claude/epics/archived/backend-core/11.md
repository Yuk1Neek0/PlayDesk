# Task: RAG layer — ingest, embeddings, pgvector retrieval

## Description

Implement the RAG retrieval layer in a new `backend/rag/` app: ingest the knowledge base, embed it, and serve top-k vector search.

## Acceptance Criteria
- [ ] A management command ingests `knowledge-base/*.jsonl` into `KnowledgeChunk` (content, category, source, lang), re-runnable idempotently.
- [ ] Embeddings via OpenAI `text-embedding-3-small` behind an injectable client interface (mockable).
- [ ] An HNSW index on `KnowledgeChunk.embedding` (migration); IVFFlat fallback documented.
- [ ] A `retrieve(query, k=5, lang=None)` function returning top-k chunks with `category`/`source` metadata.
- [ ] pytest covers ingest + retrieval with the embedding client mocked (no API calls in CI).

## Technical Details
- All code under `backend/rag/`; do NOT edit `config/urls.py`/`config/settings.py` — report the `INSTALLED_APPS` entry + any settings needed.
- The embedding client interface is reused by `search_knowledge_base` in task 003.

## Dependencies
Foundation KB content + `KnowledgeChunk` model.
