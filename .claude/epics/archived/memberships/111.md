# Task: Admin frontend — Membership card + /admin/rewards + /admin/tiers + public QR tier badge

## Description

All the user-visible surface for the epic. Adds a "Membership" card to the existing customer detail page (balance, tier, transactions, adjust + redeem modals), two new admin pages for rewards / tiers CRUD, and a small SSR'd tier badge in the public QR landing-page header.

The QR tier badge is the only non-admin surface in v4 and the only place that reads the `pd_customer` cookie server-side for memberships data.

## Acceptance Criteria
- [ ] `frontend/src/app/admin/customers/[id]/page.tsx` renders a new `<MembershipSection />` below the existing visit-history table.
- [ ] The membership card shows: large current balance, tier name + perks text (or "No tier yet" with the points-to-next-tier hint), recent-transactions table (date, delta with +/- sign, source label, reference excerpt).
- [ ] "Adjust" button opens a modal with a `delta` number input and a required `reason` textarea; submits to `POST /api/admin/customers/{id}/adjust-points/` and refreshes the section on success.
- [ ] "Redeem" dropdown lists only rewards where `cost_points <= balance`; selecting one triggers a confirmation step + `POST /api/admin/customers/{id}/redeem/`; refreshes on success; shows the 409 error inline on failure.
- [ ] `frontend/src/app/admin/rewards/page.tsx` — CRUD list (table + create / edit drawer). Fields: name, description, cost_points, enabled.
- [ ] `frontend/src/app/admin/tiers/page.tsx` — CRUD list. Fields: name, min_lifetime_points, perks_text, position.
- [ ] `frontend/src/app/qr/[slug]/page.tsx` extended to read the `pd_customer` cookie SSR-side and render a small tier badge in the header for identified visitors. Anonymous visitors see no badge.
- [ ] All new UI honours `prefers-reduced-motion` (no slide animations).
- [ ] Existing visit-history layout on the customer detail page is unchanged.
- [ ] `npm run lint && npm run typecheck && npm run build` all clean.

## Technical Details

- Files added: `frontend/src/components/admin/membership-section.tsx`, `frontend/src/components/admin/adjust-points-modal.tsx`, `frontend/src/components/admin/redeem-modal.tsx`, `frontend/src/app/admin/rewards/page.tsx`, `frontend/src/app/admin/tiers/page.tsx`.
- File extended: `frontend/src/app/qr/[slug]/page.tsx` (SSR cookie read + tier-badge render). The cookie's value maps to a customer id; the new SSR fetch hits `GET /api/admin/customers/{id}/membership/` with the backend's service token — or a new lightweight `GET /api/qr/tier/?customer_id=N` endpoint scoped to badge-only data (decided in implementation).
- Reuse `pd-card`, `pd-button`, `pd-modal` design tokens — no new CSS file.
- The Membership card uses optimistic refresh on success of the adjust / redeem mutations (re-fetch the composite endpoint after the mutation resolves).
