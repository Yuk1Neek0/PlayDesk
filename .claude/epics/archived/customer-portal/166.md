# Task: Customer auth foundation

## Description

The auth layer that every other v7 task builds on. Adds two new models (`CustomerOTP`, `CustomerLoginAttempt`), three new endpoints (request-code, verify-code, logout), and `CustomerSessionMiddleware` that sets `request.customer` on every request. OTP delivery uses the existing v4 outbound SMS adapter.

## Acceptance Criteria

- [ ] `core.CustomerOTP(phone, code, expires_at, attempts, created_at)` model + migration. `code` is 6 digits, `expires_at` is 10 min from creation, `attempts` starts at 0.
- [ ] `core.CustomerLoginAttempt(phone, store, success, ip_address, created_at)` model + migration â€” append-only audit log.
- [ ] `POST /api/customer-auth/request-code/` body `{phone, store_slug}`:
   - Invalidates any existing `CustomerOTP` for that phone+store.
   - Generates a new 6-digit code, persists, sends via v4 outbound `OutboundChannelAdapter` with template `customer_otp`.
   - Rate-limited: 1 request per phone per 60s, 5 per phone per hour. Use Django cache for the rate-limit window.
   - Returns `{request_id}` (the OTP row's id).
- [ ] `POST /api/customer-auth/verify-code/` body `{phone, code, store_slug}`:
   - Looks up the OTP row, increments `attempts`. If `attempts > 5` â†’ invalidate + 429.
   - If `code` matches and not expired â†’ find-or-404 `Customer` by `phone` + `store_slug`. Sign a `pd_customer_session` cookie with Django signed cookies: payload `{customer_id, store_id, exp}` (exp = now + 30 days). Set HttpOnly + Secure + SameSite=Lax. Mark the OTP row as `used_at=now()`.
   - On success, write a `CustomerLoginAttempt(success=True)` row.
   - Returns `{customer: {id, name}}` on success, 401 otherwise.
   - If the customer doesn't exist for that phone+store, still create the CustomerLoginAttempt(success=False) and return 401 â€” don't leak whether the phone is registered.
- [ ] `POST /api/customer-auth/logout/` â†’ clears the cookie. Always 200.
- [ ] `core.middleware.CustomerSessionMiddleware`:
   - Reads `pd_customer_session` cookie, validates Django signature, parses payload.
   - If valid + non-expired + `store_id == request.store.id` â†’ `request.customer = Customer.objects.get(pk=customer_id)`.
   - If signature invalid OR expired OR store mismatch â†’ `request.customer = None` and delete the cookie in the response.
   - Registered in `MIDDLEWARE` after `CurrentStoreMiddleware`.
- [ ] `customer_otp` template added to `core.OutboundTemplate` seed: `"Your PlayDesk verification code: {code}. Expires in 10 minutes."`
- [ ] Tests in `tests/test_customer_auth.py`:
   - request-code â†’ SMS sent (mock outbound), OTP row created
   - request-code rate-limit: 2nd within 60s â†’ 429
   - request-code rate-limit: 6th within 1h â†’ 429
   - verify-code happy path â†’ cookie set, customer returned
   - verify-code wrong code â†’ attempts++; after 5 attempts â†’ 429 + OTP invalidated
   - verify-code expired â†’ 401
   - verify-code unknown phone â†’ 401 + CustomerLoginAttempt(success=False) recorded
   - verify-code with store_slug mismatch (phone exists at Flagship, verified against North) â†’ 401
   - middleware: valid cookie â†’ request.customer set
   - middleware: cookie with stale store_id â†’ request.customer is None
   - middleware: tampered signature â†’ request.customer is None + cookie cleared
- [ ] Existing backend test suite passes unchanged.

## Technical Details

- Use `django.core.signing.TimestampSigner` for cookie signing â€” simpler than rolling JWTs, no new pip dep.
- Rate limit via `django.core.cache.cache.add(key, 1, timeout=60)` returning False = limit hit.
- Use the same Twilio adapter path as v4 outbound (`outbound.tasks.send_outbound_message_sync(...)` or similar â€” check existing call site).
- 6-digit code: `secrets.choice("0123456789")` Ă— 6, NOT random.random â€” codes are auth tokens.
- `attempts` is per-OTP, not per-phone. A new request-code resets it because it issues a fresh OTP row.
