# Task: Customer-scoped /api/me/ endpoints

## Description

Customer-scoped REST endpoints for profile, bookings list, reschedule, and cancel. All require `request.customer` (401 otherwise) and operate on rows the customer owns (404 otherwise). Adds `Store.cancellation_lead_hours` for the cancellation policy.

## Acceptance Criteria

- [ ] `Store.cancellation_lead_hours: PositiveIntegerField(default=24)` + migration.
- [ ] `GET /api/me/` → `{id, name, phone, store_slug}`. 401 if not logged in.
- [ ] `PATCH /api/me/` body `{name}` → updates customer.name only. Phone is read-only. 401 / 200.
- [ ] `GET /api/me/bookings/?status=upcoming|past&limit=N&offset=M`:
   - `upcoming`: `start_at >= now() AND status NOT IN ('cancelled','completed')`, order by `start_at` asc.
   - `past`: everything else, order by `start_at` desc.
   - Default limit 20, max 50.
   - Returns `{results: [...], total: N, has_more: bool}`.
   - Each result includes booking + nested resource summary (id, name, type).
- [ ] `POST /api/me/bookings/{id}/reschedule/` body `{start_at, end_at}`:
   - 404 if booking doesn't exist or `booking.customer_id != request.customer.id`.
   - Validates `start_at >= now()`, `end_at > start_at`, hours <= original duration + 50% (anti-abuse).
   - Reuses the booking-overlap check from booking-create (`EXCLUDE USING gist` will reject naturally).
   - On success: update booking row, send SMS via v4 outbound (template `booking_rescheduled`), return updated booking.
   - 409 if overlap, 400 if invalid timing.
- [ ] `POST /api/me/bookings/{id}/cancel/`:
   - 404 if not owned by request.customer.
   - If `booking.start_at - now() < store.cancellation_lead_hours` → return 409 `{error: "lead_time_violation", lead_hours: N, contact: "Please contact staff"}`.
   - On success: set `booking.status = 'cancelled'`, send SMS (template `booking_cancelled`), return `{status: "cancelled"}`.
- [ ] Outbound templates added:
   - `booking_rescheduled`: `"Your PlayDesk booking on {date_old} has been moved to {date_new}. Thank you!"`
   - `booking_cancelled`: `"Your PlayDesk booking on {date} has been cancelled. We hope to see you again soon!"`
- [ ] Tests in `tests/test_me_endpoints.py`:
   - 401 without cookie
   - 404 cross-customer (logged in as customer A, accessing customer B's booking)
   - GET /api/me/bookings/ pagination + status filter
   - reschedule happy path → SMS sent (mock outbound), booking row updated
   - reschedule overlap → 409
   - reschedule invalid timing → 400
   - cancel within lead time → 409
   - cancel outside lead time → 200, status='cancelled', SMS sent
- [ ] Existing backend test suite passes.

## Technical Details

- New file: `backend/api/views_me.py` for all `/api/me/*` endpoints. Wire URLs in `backend/api/urls.py`.
- Reuse the overlap-check pattern from `backend/api/views_public.py::create_booking` — wrap in `try/except IntegrityError` to catch the GIST exclusion constraint violation.
- For lead-time check: `lead_seconds = (booking.start_at - timezone.now()).total_seconds()`; convert store.cancellation_lead_hours to seconds.
- SMS send is fire-and-forget; if it fails, log + still return success to the customer (their booking change has already happened).
- The reschedule's "+50% duration cap" is a soft anti-abuse: a customer shouldn't be able to turn a 1-hour booking into a 6-hour booking via reschedule. Reasonable bound.
