## Technical Details

- The `useStaffSession()` hook returns `{user: {id, username, is_staff, is_superuser} | null, ready: boolean, logout: () => void}`. The shape was deliberately chosen to be drop-in compatible with `useAuth()` except for `user.role` â†’ `user.is_staff` boolean.
- Tip for the rename: `grep -rl 'useAuth\b' frontend/src/app/admin/ | xargs sed -i.bak 's/useAuth/useStaffSession/g' && grep -rl '@/lib/auth' frontend/src/app/admin/ | xargs sed -i.bak 's|@/lib/auth|@/lib/staff-session|g'`. Then audit + remove `.bak` files.
- The `user?.role === "staff"` predicate updates: there's only a few sites (~5). The visual ones are easy; the route-guard ones already get covered by `StaffSessionProvider`'s redirect, so removing the per-page check entirely is also acceptable.
- Don't remove the `AuthProvider` from `layout.tsx` until ALL admin files are migrated â€” otherwise the dev server crashes mid-migration.
