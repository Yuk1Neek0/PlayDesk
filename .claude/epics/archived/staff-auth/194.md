# Task: End-to-end coverage

## Description

Verification that the new staff auth surface works end-to-end. Backend unit tests already covered in tasks #190 and #191; this task adds the Playwright e2e + a frontend integration test that the provider + login page wire together.

## Acceptance Criteria

- [ ] `frontend/e2e/staff-auth.e2e.ts` (new):
   - **Test 1 — login flow happy path**: navigate to `/admin`, expect redirect to `/staff/login?next=/admin`. Enter `playdesk_staff` + `playdesk_staff_demo_pw`. Submit. Land on `/admin`. Verify the username chip in the nav says "playdesk_staff".
   - **Test 2 — wrong password**: same login form, enter wrong password, expect inline error "Invalid credentials" + still on the login page.
   - **Test 3 — logout**: from `/admin`, click Logout. Land on `/staff/login`. Refresh `/admin` → redirected back to login (cookie cleared).
   - **Test 4 — direct admin URL while unauthenticated**: visit `/admin/customers` without a cookie → redirect to `/staff/login?next=/admin/customers`. After login → land on `/admin/customers` (not `/admin`).
   - **Test 5 — localStorage spoofing doesn't work**: `await page.evaluate(() => localStorage.setItem("playdesk.user", JSON.stringify({name: "fake", role: "staff"})))`. Reload `/admin`. Should STILL redirect to `/staff/login` — the localStorage hack is dead.
- [ ] `frontend/src/lib/staff-session.test.tsx` (extend if task #192 didn't fully cover):
   - Provider with mocked 200 me → user resolved, no redirect.
   - Provider with mocked 401 on `/admin/foo` → router.replace called with `/staff/login?next=/admin/foo`.
- [ ] Verify the previously-existing admin e2e tests (admin.e2e.ts, multi-location.e2e.ts, v4-cross-slice.e2e.ts) still pass — they should, because they probably create a Django superuser fixture in a beforeAll hook. If they don't, add the staff-login step.
- [ ] CI workflow updates: ensure the Playwright job runs `staff-auth.e2e.ts`. The existing pattern probably picks up all `*.e2e.ts` files automatically.
- [ ] Existing 687-test backend suite passes.
- [ ] `npm run lint && npm run typecheck && npm test && npm run build` clean.

## Technical Details

- Use the seed_data fixture from task #190 — the seeded `playdesk_staff` user is the canonical test login.
- Playwright fixture file: check `frontend/playwright.config.ts` and `frontend/e2e/fixtures/` (if any exists) for the existing setup pattern.
- The "localStorage spoof" test is the headline assertion — it's the failure mode the entire epic exists to fix. Make it a screenshot-on-failure too.
- The "wrong password" test should also assert that the 5-attempt rate limit triggers after the 6th wrong attempt within 15 min (429 response). This requires resetting the cache between tests OR using a unique username per test run.
