# Task: adminGetBusinessMetrics typed client + payload type

## Description

Frontend API client wrapper around the new `/api/admin/metrics/business/` endpoint. One function + one type. Lives alongside the existing `adminGetCustomer` / `adminListSegments` etc. in `frontend/src/lib/api.ts`.

## Acceptance Criteria
- [ ] `frontend/src/lib/api.ts` exports:
   - `interface BusinessMetricsPayload { ... }` matching the endpoint shape exactly (six top-level keys with their nested shapes; see PRD).
   - `adminGetBusinessMetrics(days?: number): Promise<BusinessMetricsPayload>` — fetches `/api/admin/metrics/business/?days=<days>` (omit query param when days is undefined).
- [ ] Calls go through the existing `request()` helper (consistent error-handling, ApiError on 4xx/5xx).
- [ ] No vitest unit test needed — the existing `request()` is already tested; this is a thin wrapper.
- [ ] `npm run lint && npm run typecheck` clean.

## Technical Details

- The interface uses the same naming style as other interfaces in `api.ts` (camelCase property names matching the JSON keys exactly — Python snake_case ones, since the backend doesn't transform them).
- Nested shapes (`bookings_today.count`, `bookings_today.trend_pct_vs_yesterday: number | null`, etc.) get their own nested interfaces for readability.
- Default-arg behaviour: when `days` is omitted, the URL is `/api/admin/metrics/business/` with no query string — the backend defaults to 30.
