# Task: Phone normalisation helper + adoption in booking creation

## Description

`backend/core/phone.py::normalize_phone(raw: str, country: str = "CA") -> str | None` returns E.164 (e.g. `+14165550188`) or `None` if unparseable. Adopted by `agent_tools/tools.py::create_booking` and `api/views.py` booking endpoints so every Booking that hits the DB carries a normalised phone and (next task) is linked to a `Customer`.

## Acceptance Criteria

- [ ] `normalize_phone("+1 (416) 555-0188") == "+14165550188"`.
- [ ] `normalize_phone("garbage") is None`.
- [ ] `normalize_phone(normalize_phone(x)) == normalize_phone(x)` (idempotent).
- [ ] `agent_tools.create_booking` calls `normalize_phone` on input and rejects unparseable phones with a structured error.
- [ ] `api/views.py` booking POST also normalises on receipt.
- [ ] 5+ unit tests covering parse / invalid / idempotency / different country defaults.

## Technical Details

- Add `phonenumbers` to `backend/requirements.txt` (industry-standard library).
