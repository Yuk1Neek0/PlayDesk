# Task: Conflict-aware slot suggestions

## Description

When `check_availability` finds nothing for the requested time, populate its
`suggestions` field with 1–2 nearby alternative slots so the LLM can relay them
instead of issuing a flat refusal.

## Acceptance Criteria
- [ ] When the requested slot is unavailable, `check_availability` returns 1–2 nearby alternatives in its `suggestions` field.
- [ ] Alternatives are real, bookable slots (respect the overlap constraint).
- [ ] The agent relays the suggestions in its reply to the user.
- [ ] Tests cover the unavailable-with-alternatives and no-alternatives cases.
- [ ] CI green: `ruff check`, `ruff format`, `pytest`.

## Technical Details

- No schema change — the `suggestions` key already exists in the tool contract.
- Only modifies the `check_availability` tool implementation.
- Disjoint code path — conflict-free with tasks 001, 002, 004.
