# Task: Evaluation harness

## Description

Build a replay-based evaluation harness for the agent: 10–15 labeled
conversations and a runner that replays them against the agent loop, asserting
correct tool use, booking outcome, and an on-topic final message. Outputs
per-case pass/fail plus aggregate accuracy.

## Acceptance Criteria
- [ ] 10–15 labeled cases authored per `docs/contracts/eval-format.md`, covering `should_book` / `should_clarify` / `should_refuse` / `should_search_kb`.
- [ ] A replay runner executes each case against the agent loop.
- [ ] Each case asserts tool use, booking outcome, and on-topic final reply.
- [ ] Runner prints per-case results and an aggregate accuracy figure.
- [ ] LLM calls are mocked in tests — runs in CI without secrets.
- [ ] CI green: `ruff check`, `ruff format`, `pytest`.

## Technical Details

- Follow the frozen `docs/contracts/eval-format.md` for case structure.
- Reuse the existing agent loop; do not modify it.
- Disjoint code path — conflict-free with tasks 002, 003, 004.
