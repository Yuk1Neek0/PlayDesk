# Task: twilio_voice_status_callback view + missed-call Conversation row

## Description

Twilio fires a separate webhook on call status events — `completed`, `no-answer`, `busy`, `failed`. The voice scaffold should capture missed/failed calls too so the admin filter shows attempted-but-missed calls, not just answered ones.

## Acceptance Criteria
- [ ] `backend/api/webhooks_twilio.py` exports `twilio_voice_status_callback(request)` view.
- [ ] `POST /api/webhooks/twilio/voice/status/` registered in `backend/api/urls.py`.
- [ ] Verifies signature via the shared helper.
- [ ] Parses `CallStatus` (`completed`, `no-answer`, `busy`, `failed`, `canceled`) and `From`.
- [ ] On `CallStatus in ('no-answer', 'busy', 'failed', 'canceled')`:
   - Creates a `Conversation(channel='phone', customer_identifier=normalized_phone, status='abandoned')` row.
   - Links to Customer the same way as task 002's webhook.
   - Does NOT create a row if `CallStatus='completed'` (the answer-time webhook in task 002 already did).
- [ ] Returns 200 with an empty body (Twilio doesn't expect a payload on status callbacks).
- [ ] Returns 403 on tampered signature.
- [ ] Returns 503 on missing `TWILIO_AUTH_TOKEN`.
- [ ] Tests in `tests/test_voice_status_callback.py`:
   - `CallStatus=no-answer` → Conversation row with status='abandoned'.
   - `CallStatus=completed` → no new row written (the answer-time webhook handles it).
   - Tampered signature → 403, no row.
   - Missing token → 503.

## Technical Details

- Uses the existing `Conversation.status='abandoned'` choice if it exists; otherwise defaults to `'completed'` and adds a NOTE comment that abandonment tracking is a follow-on.
- Twilio needs the status-callback URL configured on the phone-number's voice settings (operator concern, documented in the v5 implementation-plan doc in task 004).
- The same Twilio call produces TWO webhooks: answer-time (task 002) AND a completion status callback. This view skips the `completed` case to avoid double-rows.

## Dependencies

- Task 002 (the answer-time view exists and we need to coordinate the no-double-write rule).
