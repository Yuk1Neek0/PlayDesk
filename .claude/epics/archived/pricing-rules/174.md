# Task: Booking pricing fields

## Description

Three additive fields on `Booking` for freezing quote data: `total_amount`, `rule_snapshot`, `refund_amount`. Plus a data migration that backfills existing bookings to `total_amount = resource.hourly_rate * hours`, `rule_snapshot = []`, `refund_amount = 0`. Parallelizable with task 173 — they touch different files.

## Acceptance Criteria

- [ ] `Booking.total_amount: DecimalField(max_digits=10, decimal_places=2, null=True)` — nullable initially for backfill, made non-null in a follow-up step within the same migration via `RunPython`.
- [ ] `Booking.rule_snapshot: JSONField(default=list)` — stores `[{label, amount, rule_id}]` line items from `Quote`.
- [ ] `Booking.refund_amount: DecimalField(max_digits=10, decimal_places=2, default=Decimal("0.00"))` — placeholder field; v9 billing-payments will populate.
- [ ] Migration plan (two-file approach is fine OR one combined; whichever the agent finds cleaner):
   1. `000X_booking_pricing_fields.py` — add the fields nullable.
   2. `000Y_backfill_booking_total_amount.py` — `RunPython` that loops `Booking.objects.all()` and sets `total_amount = resource.hourly_rate * Decimal(hours)` where `hours = (end_at - start_at).total_seconds() / 3600`.
   3. (Optional) `000Z_booking_total_amount_required.py` — `AlterField` to set `null=False`.
- [ ] `BookingSerializer` (or wherever bookings serialize for the API) exposes `total_amount`, `rule_snapshot`, `refund_amount` as read-only fields. Admin booking detail and v7 portal upcoming-bookings list both pick these up automatically.
- [ ] Tests in `tests/test_booking_pricing_fields.py`:
   - Existing bookings have non-null total_amount after migration.
   - New bookings can be created with `total_amount=Decimal("123.45")`.
   - `rule_snapshot` round-trips a list-of-dicts.
   - `refund_amount` defaults to `Decimal("0.00")`.
- [ ] Existing backend test suite passes (all booking-create tests should still pass — task 176 hooks the quote computation in; until then, booking-create writes a stubbed total_amount from `resource.hourly_rate * hours`).

## Technical Details

- The backfill uses `Decimal((end_at - start_at).total_seconds() / 3600)` — careful: `total_seconds()` returns float, must wrap in Decimal AFTER converting. Pattern: `Decimal(str(hours))` or split into hours/minutes integers.
- Run the backfill in a transaction with chunking (every 1000 rows commit) only if necessary. Current dataset is small, single transaction is fine.
- After this task: booking-create can stub `total_amount = resource.hourly_rate * Decimal(hours)` so the field is always populated even before task 176 wires the real quote computation.
