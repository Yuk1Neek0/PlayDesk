# Task: Pricing-rule admin UI + booking-detail rule snapshot

## Description

The admin pricing-rule manager and the customer/admin booking surfaces that consume the new fields. Closes the v8 epic loop — staff can configure rules, customers see line items, admin sees what was charged.

## Acceptance Criteria

- [ ] `POST/GET/PATCH/DELETE /api/admin/pricing-rules/` (DRF ModelViewSet):
   - Store-scoped via `request.store` (mirrors v4 patterns).
   - Server-side validation calls `strategy.validate_params(params)` for the rule's type → 400 with clear errors on shape mismatch.
- [ ] `frontend/src/app/admin/pricing/page.tsx`:
   - Table of rules: name · type · priority (drag-handle) · enabled toggle · stackable · edit · delete.
   - "New rule" button → opens RuleForm.
   - Reorder updates `priority` via PATCH.
- [ ] `frontend/src/app/admin/pricing/RuleForm.tsx`:
   - Type dropdown drives a dynamic params form (one form per rule_type).
   - Field validation client-side mirrors `params_schema`.
   - Save → POST or PATCH `/api/admin/pricing-rules/`.
- [ ] `frontend/src/app/admin/pricing/QuoteSandbox.tsx`:
   - Pick resource + start_at + end_at + (optional) customer + tier → call `/api/quote/` → render line items.
   - Does not save anything.
- [ ] `frontend/src/app/s/[slug]/book/BookingPage.tsx`:
   - Below the time picker, render the quote line items returned from `POST /api/quote/` as a small table: "Base · $80.00 / Member discount · -$12.00 / Total · $68.00".
- [ ] Admin booking detail (`frontend/src/app/admin/bookings/[id]/page.tsx` or wherever):
   - Show `total_amount` prominently.
   - Show `rule_snapshot` as a collapsed line-items table.
- [ ] Tests:
   - `tests/test_pricing_rules_api.py`: CRUD happy paths + 400 on bad params + 404 cross-store.
   - Frontend smoke test (vitest): RuleForm renders the right fields for each rule_type.
- [ ] `npm run lint && npm run typecheck && npm test && npm run build` clean.

## Technical Details

- The dynamic params form: a switch on `rule_type` rendering 5 different form fragments. Don't over-abstract — explicit is fine.
- Drag-handle for priority: use a small lib (e.g., `@dnd-kit/sortable`) or a simpler up/down arrow if you want zero new deps. The PRD doesn't require drag-drop specifically.
- For the QuoteSandbox: a side panel or modal. Quick iteration loop for admin testing.
- The admin booking-detail page may not exist yet as a separate route — check whether bookings have a detail page or only inline list view. If only list view, add the rule_snapshot to the list row's expandable detail.
