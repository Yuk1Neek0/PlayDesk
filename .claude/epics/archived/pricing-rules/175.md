# Task: Quote engine + API + agent tool

## Description

The pricing engine itself + the public quote endpoint + the agent tool update. After this task, the booking page (and the agent) can show real quotes; the actual freezing onto Booking happens in task 176.

## Acceptance Criteria

- [ ] `pricing.engine.Quote` dataclass:
   - `base_amount: Decimal`, `line_items: list[QuoteLineItem]`, `total_amount: Decimal`, `rule_snapshot: list[dict]`.
   - Method `to_dict()` for JSON serialization.
- [ ] `pricing.engine.QuoteLineItem` dataclass:
   - `label: str`, `amount: Decimal` (signed), `rule_id: Optional[int]` (None for base).
- [ ] `pricing.engine.compute_quote(resource, start_at, end_at, customer=None) -> Quote`:
   1. Compute `hours = Decimal((end_at - start_at).total_seconds() / 3600)`, rounded to 4dp internally.
   2. `base = resource.hourly_rate * hours`. Append base line item.
   3. Resolve `customer_tier` from `customer` (via v4's `tier_for(customer)`).
   4. Load `PricingRule.objects.filter(store=resource.store, enabled=True).order_by("priority", "id")`. Filter to those where `applies_to_resource is None OR applies_to_resource == resource`.
   5. For each rule:
      - If a non-stackable rule has already fired and this rule is also non-stackable → skip.
      - `strategy = RULE_REGISTRY[rule.rule_type]`.
      - If `strategy.applies(rule, ctx)` is False → skip.
      - `adj = strategy.compute(rule, ctx, running_total)`.
      - Append line item `{label: rule.name, amount: adj, rule_id: rule.id}`. Update running total.
      - If `not rule.stackable` → set the "non-stackable fired" flag.
   6. Floor running total at 0.
   7. Round final to 2dp.
   8. `rule_snapshot` = serialized line items.
- [ ] `POST /api/quote/` (public, no auth):
   - Body: `{resource_id, start_at, end_at, customer_id?}`.
   - Validates resource belongs to `request.store`. Loads customer if customer_id given.
   - Calls `compute_quote(...)`, returns `Quote.to_dict()`.
- [ ] `agent_tools.tools.check_availability` returns each slot with a `quoted_price` field (calls `compute_quote` for that resource+slot+anonymous customer). Existing fields preserved.
- [ ] `agent_tools.tools.get_resource_details` returns `hourly_rate` (unchanged) plus `display_price_text` (e.g., "from $50/hr" if there's a discount rule that could apply, else "$50/hr").
- [ ] Tests in `pricing/tests/test_engine.py`:
   - Zero rules → returns base only.
   - Single peak_hours rule that applies → 20% surcharge math correct.
   - Single member_tier discount → math correct.
   - Stackable peak + member → both apply.
   - Non-stackable peak + stackable member: peak fires first, member also fires (member is stackable).
   - Non-stackable Cheap-Tuesday + non-stackable member: only first one fires.
   - Negative total floored at 0.
   - 50 rules → < 50ms (perf assertion using `time.perf_counter`).
   - `bracket_rate`: 4 hours with brackets [{max_hours: 2, rate: 50}, {max_hours: null, rate: 30}] → 100 + 60 = $160.
- [ ] Test in `tests/test_quote_api.py`: API returns correct shape + customer context applied.
- [ ] Test in `agent_tools/tests/test_check_availability_quoted.py`: tool returns `quoted_price` correctly after a peak_hours rule.
- [ ] Existing tests pass.

## Technical Details

- Performance: avoid N+1 queries. Single `PricingRule.objects.filter(store=...).select_related("applies_to_resource")` outside the loop. The customer-tier lookup is one extra query.
- The "non-stackable" logic: track `non_stackable_fired: bool`. If a non-stackable rule fires, subsequent non-stackable rules are skipped.
- `compute_quote` is pure (no DB writes). It's called from the API endpoint AND from booking-create (task 176) AND from the agent tool.
- Floor at zero AFTER summing all adjustments. Don't floor per-rule.
- For decimal precision: do all math at higher precision (`getcontext().prec = 8` inside the engine), round to 2dp at the end.
