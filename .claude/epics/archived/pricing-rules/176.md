# Task: Wire booking-create to freeze the quote

## Description

Hook `compute_quote` into the public booking-create endpoint so every new booking gets a real frozen `total_amount` + `rule_snapshot`. Adds optimistic-concurrency via `expected_total_amount` to catch the rare race where a rule changes between quote and submit.

## Acceptance Criteria

- [ ] `backend/api/views_public.py::create_booking` (or equivalent):
   - After resource + slot validation, call `compute_quote(resource, start_at, end_at, customer=customer_or_none)`.
   - Write `booking.total_amount = quote.total_amount`, `booking.rule_snapshot = [li.to_dict() for li in quote.line_items]`.
   - If request body includes `expected_total_amount` and it differs from `quote.total_amount` by more than $0.01 → return 409 `{error: "quote_changed", new_quote: quote.to_dict()}`. Booking NOT created.
- [ ] Frontend booking page (`frontend/src/app/s/[slug]/book/BookingPage.tsx`):
   - On slot selection, fetch `POST /api/quote/` and display line items below the picker.
   - On submit, include `expected_total_amount: quote.total_amount` in the booking-create body.
   - On 409 → re-show the quote with the new total + "Price changed, please confirm" message.
- [ ] Tests in `tests/test_booking_create_quote.py`:
   - Booking-create with no rules → total_amount = resource.hourly_rate * hours.
   - Booking-create with a peak_hours rule → total_amount reflects the surcharge.
   - Booking-create with `expected_total_amount` matching → 201.
   - Booking-create with `expected_total_amount` mismatching → 409 with new quote payload.
   - Booking-create with no `expected_total_amount` → 201 (backwards compat — old clients that don't send it just accept whatever the server computed).
   - rule_snapshot is JSON-serializable and round-trips.
- [ ] Existing tests pass.

## Technical Details

- The expected_total_amount comparison uses `Decimal` not float. Coerce input → Decimal → compare with `abs(...) > Decimal("0.01")`.
- `customer_or_none` for the quote: if the booking request includes a cookie-resolved customer (existing v3 one-qr pattern), pass it. Else None.
- The 409 returns the new quote so the frontend can show "price changed, here's the new total — confirm?" without an extra round-trip.
- Don't break existing booking-create callers (the e2e tests, the agent's `create_booking` tool). The tool calls also go through this endpoint or a sibling; ensure the agent tool's behavior stays correct.
- Verify the v7 customer-portal reschedule endpoint (task 167) also re-computes the quote on reschedule — if not handled there, add a follow-up note. v7's task 167 is being written without v8 knowledge — it'll only know about pricing on merge. **Document this in the PR description as a known follow-on.**
