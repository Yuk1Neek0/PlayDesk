# Task: API & streaming contracts

## Description

Author the REST and streaming contracts that let the frontend and backend build concurrently against a frozen interface.

## Acceptance Criteria
- [ ] `docs/contracts/openapi.yaml` is a valid OpenAPI 3.1 spec covering every planned endpoint: resources list + availability, booking CRUD (`409` on overlap), conversation messages (streaming), admin conversations/bookings.
- [ ] Request/response schemas include the forward-compatible fields: `Booking.status` with `pending_payment`, and a `check_availability`-style response carrying an `available` array and a `suggestions` array.
- [ ] `docs/contracts/sse-protocol.md` defines the SSE event protocol: event names and JSON payloads for `token`, `tool_call_start`, `tool_call_end`, `done`, `error`.
- [ ] TS type generation is documented/wired (e.g. an `openapi-typescript` command) so the frontend can consume generated types.
- [ ] Spec validates with a standard OpenAPI validator.

## Technical Details
- Spec authored from `cosready_demo_dev_plan.md` §1.1.
- Keep schemas the single source of truth; both stacks generate from them.
- All files live under `docs/contracts/` — no conflict with backend or frontend code.

## Dependencies
None — authored from the dev plan; runs concurrently.
