# Task: Frontend scaffold & page shells

## Description

Create the Next.js + Tailwind frontend skeleton with route shells for the three product pages and dummy auth. Pages render static placeholders — real wiring is Wave 1+.

## Acceptance Criteria
- [ ] `frontend/` is a Next.js (App Router) project with Tailwind configured (defaults only — no custom design system).
- [ ] Route shells exist and render: `/` (manual booking), `/chat` (AI front desk), `/admin` (staff dashboard).
- [ ] Dummy one-click login (no passwords, no email verification).
- [ ] `package.json` defines `lint`, `typecheck` (`tsc --noEmit`), `test` (vitest), and `build` scripts.
- [ ] `npm run lint`, `npm run typecheck`, `npm test`, and `npm run build` all pass.
- [ ] At least one trivial vitest test exists so the CI `frontend` job has something to run.

## Technical Details
- App Router under `frontend/app/`; shared layout with nav between the three pages.
- TypeScript strict mode; this is where OpenAPI-generated types (task 004) will land later.
- Keep components placeholder-only; no API calls yet.

## Dependencies
None — independent of the backend; runs concurrently with all other tasks.
