# Task: twilio_whatsapp_webhook view + URL conf + tests

## Description

Public webhook view that Twilio's WhatsApp service POSTs inbound messages to. Verifies the signature using the shared helper, normalises via `TwilioWhatsAppAdapter`, routes through the agent loop, returns TwiML.

## Acceptance Criteria
- [ ] `backend/api/webhooks_twilio.py` exports `twilio_whatsapp_webhook(request)` view.
- [ ] `POST /api/webhooks/twilio/whatsapp/` registered in `backend/api/urls.py`.
- [ ] Signed valid payload → 200 + TwiML containing the agent's reply.
- [ ] Tampered signature → 403.
- [ ] `TWILIO_AUTH_TOKEN` unset → 503 with `{"error":"not_configured"}` body (matches the existing SMS webhook).
- [ ] The view creates or reuses a `Conversation` with `channel='whatsapp'`, customer_identifier = the normalised phone (after stripping `whatsapp:` prefix).
- [ ] Tests in `tests/test_whatsapp_webhook.py`:
   - Signed valid POST → 200 + TwiML body contains an agent reply token.
   - Tampered signature → 403, no Conversation row written.
   - Missing `TWILIO_AUTH_TOKEN` → 503, body `{"error":"not_configured"}`.
   - Two messages in sequence from the same `whatsapp:+E.164` reuse the same Conversation row.

## Technical Details

- The webhook is structurally identical to `twilio_sms_webhook` — copy the body, swap the adapter, swap the URL path. Adapter polymorphism makes this near-mechanical.
- The agent-loop dispatch uses the same `AgentLoop.run(conversation, user_content)` shape as the SMS webhook; no agent changes needed.
- TwiML response: `Content-Type: application/xml`; body is `<Response><Message>...</Message></Response>`.
