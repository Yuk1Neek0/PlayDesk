# Task: Admin CRUD — list / create / patch (reorder) / delete + analytics

## Description

Staff endpoints for managing QR actions plus the analytics endpoint that powers the admin card row.

## Acceptance Criteria

- [ ] `GET /api/admin/qr-actions/` lists the current store's actions ordered by `position`.
- [ ] `POST /api/admin/qr-actions/` creates a new action; if no `position` given, appends to the end.
- [ ] `PATCH /api/admin/qr-actions/{id}/` accepts `label`, `target_url`, `reward_points`, `enabled`, and `position`. Reordering compacts positions atomically inside `transaction.atomic()`.
- [ ] `DELETE /api/admin/qr-actions/{id}/` removes and re-compacts positions.
- [ ] `GET /api/admin/qr-analytics/?days=7` returns `{ scans, clicks, engagement_rate, per_action: [...] }` with null-safe math (zero rows → zeros, not NaN).
- [ ] All endpoints reject non-staff with 403.
- [ ] 7+ tests covering happy paths + reorder atomicity + null-safe analytics.

## Technical Details

- `backend/api/qr_admin.py`.
- Reorder logic: load all actions for the store ordered by `position`, splice the changed action into its new position, re-write positions in a single transaction.
