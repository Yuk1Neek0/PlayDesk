# Task: Seed a second store + cross-store isolation e2e test

## Description

The visible payoff for the epic. Extend `seed_data.py` with a second store ("PlayDesk North · Toronto"), its resources, and its QR actions. Then a Playwright e2e test that exercises the cross-store isolation invariant end-to-end: switch stores in admin, see different data; book at one store, verify it doesn't show in the other.

## Acceptance Criteria
- [ ] `backend/core/management/commands/seed_data.py` extended:
   - Idempotently creates a second `Store(name="PlayDesk North · Toronto", slug="playdesk-north", timezone="America/Toronto")`.
   - Seeds 2 PS5 stations + 1 private room for the new store.
   - Seeds the default 4 QR actions for the new store (review, Instagram, WeChat, WiFi).
   - Existing flagship store's seed is unchanged.
   - Re-running `python manage.py seed_data` produces no duplicates (use `get_or_create` for each row).
- [ ] `frontend/e2e/multi-location.e2e.ts` (new Playwright test):
   - Logs in as Staff via the demo button.
   - Asserts the `<StoreSwitcher />` is visible in the admin nav with TWO chips ("PlayDesk Downtown" + "PlayDesk North", or whatever the seed names are).
   - Selects the North store; asserts the admin page re-fetches and shows zero bookings (because North is empty).
   - In a separate browser context (anonymous), navigates to `/s/playdesk-north/book` and books a session.
   - Back in the admin browser: still on North, refreshes — sees the new booking.
   - Switches to Flagship — sees ZERO bookings from the new North customer (cross-store isolation).
- [ ] Run the full Playwright suite — every existing test still passes (the redirect from `/` is transparent; admin switcher being visible is additive).
- [ ] Backend `pytest` still passes — the seed-data extension introduces no test regressions.
- [ ] `ruff check` + `ruff format --check` clean.
- [ ] Document the second store in CLAUDE.md's "Quickstart" section (one line: "After `docker compose up`, you can switch between PlayDesk Downtown and PlayDesk North in the admin nav").

## Technical Details

- The North store's slug is `playdesk-north`. URL: `/s/playdesk-north/book`. QR landing: `/qr/playdesk-north` (existing v3 one-qr route — already store-aware).
- Idempotent seeding uses `get_or_create` for the store and each resource; existing flagship seed already follows this pattern.
- The e2e test is the FIRST test in the integrity suite that genuinely tests multi-store behaviour. It's the kind of test that catches "I forgot to scope this query" regressions in future PRs.
- The test takes ~10-15s end-to-end (booking flow + admin nav). That's fine in the integrity suite's wall-time budget.
