# Task: Template registry + SafeFormatter + enqueue_message public API

## Description

The template surface and the public entry point. Four initial templates (booking_confirmation, reminder_24h, no_show_followup, booking_thank_you) keyed by `template_key`, rendered against `customer.locale_pref`. The `enqueue_message()` function is the single Python API that signals (task 004) and the `campaigns` slice call.

## Acceptance Criteria
- [ ] `backend/outbound/templates.py` exports `TEMPLATES: dict[str, tuple[str, str]]` with one `(en, zh)` tuple per `template_key`.
- [ ] Initial entries: `booking_confirmation`, `reminder_24h`, `no_show_followup`, `booking_thank_you` â€” all written in plain, reviewer-ready prose (no placeholders).
- [ ] A registry entry for `campaign` (used by the `campaigns` slice via `enqueue_message(..., template_key="campaign")`) where the body is passed verbatim â€” no template lookup, no rendering.
- [ ] `SafeFormatter` renderer raises `KeyError` on missing keys at render time (not silently empty-string substituted). Tested.
- [ ] `enqueue_message(customer, template_key, context, scheduled_for=None, channel='sms', reference='')` lives at `backend/outbound/api.py`:
  - Picks the correct locale template (defaults to `en`).
  - Renders the body against `context`.
  - Writes one `OutboundMessage` row with the rendered body.
  - Returns the row.
  - **Idempotence**: if `reference` is non-empty and a row exists with the same `(reference, template_key)` in `status in ('queued', 'sent')`, returns the existing row without writing a new one.
- [ ] Tests: render in both locales; render with a missing key raises `KeyError`; idempotent enqueue returns the same row; non-idempotent (empty reference) enqueue always writes.
- [ ] `campaigns` slice's send-stub fallback can import `enqueue_message` without exceptions when this task lands.

## Technical Details

- File: `backend/outbound/templates.py`.
- `SafeFormatter` subclasses `string.Formatter` and overrides `get_field` to raise `KeyError(f"template {template_key!r} missing context key {field_name!r}")` on missing fields.
- `enqueue_message` is a thin wrapper â€” no I/O outside the single `OutboundMessage.objects.create()`.
- Idempotence query: `OutboundMessage.objects.filter(reference=reference, template_key=template_key, status__in=['queued','sent']).first()`. Indexed via the composite index from task 001.
