# Task: Deposit + refund configuration

## Description

Schema + helpers for "how much do we charge as deposit" and "how much do we refund on cancel". Pure schema + a couple of pure functions; no Stripe calls in this task.

## Acceptance Criteria

- [ ] Store augmentation:
   - `deposit_mode: CharField(choices=[("none","none"),("percentage","percentage"),("fixed","fixed")], default="none")`
   - `deposit_value: DecimalField(max_digits=10, decimal_places=2, default=Decimal("0.00"))` — percentage (0-100) OR fixed amount, depending on mode.
   - `refund_matrix: JSONField(default=lambda: [{"min_hours": 48, "refund_pct": 100}, {"min_hours": 24, "refund_pct": 50}, {"min_hours": 0, "refund_pct": 0}])`
- [ ] Resource augmentation:
   - `deposit_override_mode: CharField(choices=[...], null=True, blank=True)` (same choices as store + null).
   - `deposit_override_value: DecimalField(max_digits=10, decimal_places=2, null=True, blank=True)`.
- [ ] Migrations:
   - `core/migrations/000N_store_deposit_config.py`
   - `core/migrations/000O_resource_deposit_override.py`
- [ ] `billing.helpers.calc_deposit(store, resource, total_amount) -> Decimal`:
   - If `resource.deposit_override_mode` is not None → use override.
   - Else use store defaults.
   - If mode is "none" → return Decimal("0.00").
   - If mode is "percentage" → `min(total_amount, total_amount * value / 100)`.
   - If mode is "fixed" → `min(total_amount, value)`.
   - Always: round to 2dp.
- [ ] `billing.helpers.calc_refund_amount(store, booking, now=None) -> Decimal`:
   - `lead_hours = (booking.start_at - (now or timezone.now())).total_seconds() / 3600`
   - Find first matrix row where `lead_hours >= row["min_hours"]`.
   - `refund_pct = row["refund_pct"]` (default 0 if no match).
   - Returns `booking.deposit_amount * refund_pct / 100`, rounded to 2dp.
- [ ] Admin settings page (extends 179's settings page): editor for `deposit_mode`, `deposit_value`, and `refund_matrix` (JSON textarea is acceptable for v9; structured editor is nice-to-have).
- [ ] Tests in `billing/tests/test_helpers.py`:
   - calc_deposit: each mode + override + capped at total.
   - calc_refund_amount: 100% > 48h, 50% at 36h, 0% at < 24h, edge cases (exactly 48h boundary).
   - refund_matrix JSON shape validation on save.
- [ ] Existing backend test suite passes.

## Technical Details

- The matrix is sorted DESC by `min_hours` for lookup. If the JSON is saved out-of-order, sort in the helper, don't rely on save order.
- Use `Decimal` not float throughout. Quantize at the end to 2dp.
- For the JSON editor in admin: a textarea with client-side JSON.parse() + error display is fine. Server-side validation is the source of truth.
- `deposit_value` semantically depends on `deposit_mode` — validate at save time that percentage values are 0-100.
