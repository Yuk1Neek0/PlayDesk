# Task: Stripe webhook receiver

## Description

`POST /api/stripe/webhook/`: signature-verifies Stripe events, persists raw events to `WebhookEvent` (audit + replay-safe), handles 4 event types idempotently (`payment_intent.succeeded`, `payment_intent.payment_failed`, `charge.refunded`, `account.updated`).

## Acceptance Criteria

- [ ] `POST /api/stripe/webhook/` (CSRF-exempt):
   - Read raw body + `Stripe-Signature` header.
   - Verify signature via `stripe.Webhook.construct_event(payload, sig_header, STRIPE_WEBHOOK_SECRET)`. On failure → 400.
   - Idempotency: check `WebhookEvent.objects.filter(stripe_event_id=event.id).exists()`. If yes → 200 immediately (no-op).
   - Else: create `WebhookEvent` row with raw payload. Dispatch to handler based on `event.type`. On any handler exception, log it but DO NOT return non-2xx — Stripe would retry, but the event is already persisted to WebhookEvent for manual recovery.
   - Mark `WebhookEvent.processed_at = now()` on successful handler completion.
- [ ] Handlers in `billing.webhook_handlers`:
   - `handle_payment_intent_succeeded(event)`:
     - Find `Payment` by `stripe_payment_intent_id`. Set `status="succeeded"`, `stripe_charge_id=event.data.object.latest_charge`, `stripe_event_id=event.id`.
     - Find `Booking` by `payment_intent_id` or via `Payment.booking`. If `kind=="deposit"` → `booking.payment_status="deposit_paid"`, `booking.status="confirmed"`. If `kind=="balance"` → `booking.payment_status="paid_in_full"`.
   - `handle_payment_intent_failed(event)`:
     - Find Payment, set `status="failed"`, `stripe_event_id=event.id`.
     - Booking stays in `pending_payment` (the sweep cron will clean up after 10 min).
   - `handle_charge_refunded(event)`:
     - Find Payment by `stripe_charge_id`. Create a new `Payment` row with `kind="refund"`, negative amount, `status="succeeded"`, `stripe_event_id=event.id`.
     - Update booking `payment_status` to `refunded` if full refund, else `partially_refunded`.
   - `handle_account_updated(event)`:
     - Find Store by `stripe_account_id == event.data.object.id`.
     - Update `store.stripe_charges_enabled = event.data.object.charges_enabled`.
- [ ] Tests in `billing/tests/test_webhook.py`:
   - Bad signature → 400.
   - Valid signature, new event → handler runs, WebhookEvent persisted, Payment updated.
   - Same event replayed → 200, handler does NOT re-run (idempotency).
   - payment_intent.succeeded for a deposit → Booking.payment_status='deposit_paid', status='confirmed'.
   - payment_intent.payment_failed → Payment.status='failed', Booking still pending_payment.
   - charge.refunded → refund Payment row created, Booking.payment_status='refunded'.
   - account.updated → Store.stripe_charges_enabled updated.
- [ ] Existing tests pass.

## Technical Details

- `csrf_exempt` is needed because Stripe doesn't have a CSRF token. The signature header IS the auth.
- Use `request.body` (raw bytes) for signature verification, not `request.data` (parsed JSON).
- The WebhookEvent row must be saved BEFORE the handler runs — that way, if the handler crashes, we have the event to replay later.
- For testing: `stripe.Webhook.construct_event` can be called with a payload + sig + secret. Generate a test signature with `stripe.WebhookSignature.generate_header(payload, secret)` (test utility).
- All handlers should be transaction-wrapped (`@transaction.atomic`) so a partial update isn't possible.
