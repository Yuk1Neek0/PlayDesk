# Task: Receipts + revenue dashboard + payment ledger

## Description

The surfacing work that turns payment data into business visibility: SMS + email receipts on payment events, revenue/refunds tiles on the v5 business dashboard, and a paginated `/admin/payments/` ledger.

## Acceptance Criteria

- [ ] On webhook `payment_intent.succeeded` handler (extending task 183):
   - Send SMS via v4 outbound template `payment_receipt`: `"PlayDesk: $${amount} received for your booking on {date}. Thank you!"`
   - Send email via Django's `EmailMessage` API to `customer.email` (if non-null). Subject "Payment receipt", body includes booking details + amount. Use a minimal Django template `templates/billing/email/payment_receipt.html` + plaintext alternative.
   - If `EMAIL_BACKEND` is the console backend (dev default), the email logs to stdout — that's fine.
- [ ] On webhook `charge.refunded` handler:
   - Send SMS via outbound template `refund_receipt` (similar to `booking_refunded` but post-confirmation).
   - Send email with refund details.
- [ ] Business-dashboard (v5) endpoint augmentation:
   - Add to the existing `/api/admin/metrics/` (or wherever v5's dashboard data comes from) two new fields:
     - `revenue_mtd`: `Payment.objects.filter(store=request.store, status='succeeded', kind__in=['deposit','balance'], created_at__gte=month_start).aggregate(Sum('amount'))`.
     - `refunds_mtd`: same but `kind='refund'` (will be negative → absolute value).
- [ ] v5 dashboard frontend: add two new tiles "Revenue (MTD)" + "Refunds (MTD)" displaying the new fields. Match existing tile styling.
- [ ] `frontend/src/app/admin/payments/page.tsx`:
   - Paginated table: date · booking · customer · kind · amount · status · stripe_charge_id (truncated).
   - Filters: kind, status, date range.
   - Store-scoped via v6 `adminFetch`.
   - "Sync from Stripe" button (optional v9 nice-to-have): fetches the last N payment_intents from Stripe and reconciles. Skip if time-constrained.
- [ ] `Customer.email: EmailField(blank=True)` if not already present. Migration if needed.
- [ ] Tests in `billing/tests/test_receipts_and_dashboard.py`:
   - Webhook → SMS sent (mock outbound).
   - Webhook → email sent (use Django's `mail.outbox` test pattern).
   - Dashboard endpoint returns correct revenue_mtd / refunds_mtd under store filter.
   - /admin/payments/ list endpoint returns paginated + filtered.
- [ ] `npm run lint && npm run typecheck && npm test && npm run build` clean.
- [ ] Existing tests pass.

## Technical Details

- Receipt email template: keep it minimal. HTML + plaintext alternative via `EmailMultiAlternatives`. Subject = "PlayDesk payment receipt – $XX.XX".
- If `customer.email` is empty, just log + skip the email. Don't error.
- The "Sync from Stripe" button is genuinely useful for reconciliation but secondary — implement if time, defer if not.
- The dashboard tiles use the same number-formatting helper that v5 uses (check `frontend/src/lib/formatters.ts` or similar).
- Use Django's `localize` for currency display in templates / emails.
