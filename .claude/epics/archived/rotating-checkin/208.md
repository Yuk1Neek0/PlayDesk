## Description

End-to-end Playwright test covering the full flow + edge cases. Backend unit tests are already in tasks #204 and #205; this task adds the Playwright + any frontend-unit gaps.

## Acceptance Criteria

- [ ] `frontend/e2e/rotating-checkin.e2e.ts`:
   - **Test 1 — happy path**: visit admin settings/checkin/, grab the active key, navigate to `/c-in/?k=<key>`, enter phone of the seeded customer, capture OTP via test-mode (or read from a test-only endpoint that returns the latest CustomerOTP code for the seed phone), enter OTP, see the single-match welcome card, click "I'm here", see "✓ Checked in".    - **Test 6 — admin rotate-now**: staff hits the rotate button → previous key 410s on next scan; new key works.

## Technical Details

- The e2e is more involved than v10b's because of the OTP step. Capture pattern: either an `?test_mode=1` flag on `request-otp` (DEBUG-gated) that returns `{request_id, code}` in the response, or a small `/api/_test/last-otp/?phone=...` endpoint (DEBUG-only). Pick the simpler one and document it.
- The Playwright config glob already picks up `*.e2e.ts` files.
- For the test that needs a fresh active key, the test setup can call the `rotate_checkin_keys --force --store=<slug>` management command via a test API endpoint or via Playwright's beforeEach.
- The test for "admin rotate-now" requires the test to log in as staff (v10a `playdesk_staff` user).
