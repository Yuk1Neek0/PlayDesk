# Task: Admin display + rotation control

## Description

The staff-facing surfaces. Two pages: a settings page where staff configure the rotation interval + manually rotate, and a fullscreen QR display that the store can show on a tablet at the door.

## Acceptance Criteria

- [ ] Backend admin endpoints (under v10a's `StaffOnlyMiddleware` gate at `/api/admin/*`):
   - `GET /api/admin/checkin/active-key/` → `{key, created_at, expires_at, rotation_minutes, qr_url}` where `qr_url` is the customer-facing URL: `f"{settings.SITE_URL}/c-in/?k={key}"`.
   - `POST /api/admin/checkin/rotate/` → forces a new key mint immediately. Returns the new key payload.
   - `PATCH /api/admin/checkin/settings/` body `{rotation_minutes}` → updates the current store's rotation minutes.
- [ ] `frontend/src/app/admin/settings/checkin/page.tsx`:
   - Header: "Door QR — check-in code"
   - Current key section: shows the key (masked by default, "Reveal" toggle), created-at, expires-at, countdown to next rotation.
   - Rotation-interval input: a slider or number input (1-60 min). Save → PATCH endpoint.
   - "Rotate now" button: confirms then POSTs to /rotate/.
   - "Open fullscreen display" button → `<Link href="/admin/settings/checkin/display">` opens the lobby display in a new tab.
   - Auto-refreshes the active-key payload every 30s.
- [ ] `frontend/src/app/admin/settings/checkin/display/page.tsx`:
   - Full-viewport QR (use a tiny QR lib, e.g. `qrcode` npm — small add, ~6kb minified — OR a server-rendered PNG endpoint).
   - Below the QR: "Scan to check in" text + countdown until next rotation.
   - Auto-refreshes the QR URL every `rotation_minutes / 2` seconds so it always shows the active key.
   - Designed to fit a 1024×768 tablet portrait.
   - No nav, no chrome — just QR + caption. Behind v10a's staff gate so a customer can't accidentally open it.
- [ ] If using `qrcode` npm dep: add to `frontend/package.json`. If preferring server-side rendering, add a `GET /api/admin/checkin/qr.png?k=<key>` endpoint that returns a PNG via `qrcode` python package (slightly more deps; the npm path is cleaner).
- [ ] Frontend test (vitest) for the settings page: renders the masked key, reveal toggle works.
- [ ] `npm run lint && npm run typecheck && npm test && npm run build` clean.

## Technical Details

- For QR rendering, the npm `qrcode` package is well-established and small. Render via `<QRCode value={qr_url} size={...} />` in a useEffect-driven canvas, or use the React wrapper `qrcode.react`.
- The fullscreen display page should set `metadata.robots = "noindex"` and check on mount whether staff is authenticated; if not (e.g. tablet's session expired), show "Staff: log in to refresh".
- `auto-refresh every rotation_minutes / 2 seconds` ensures the QR is at most half-stale; an aggressive 1-minute refresh would also be fine but wastes a fetch.
