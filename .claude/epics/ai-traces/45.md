# Task: Agent-loop capture — populate trace fields on every Message write

## Description

In `backend/agent/loop.py`, populate the new `Message` fields on every assistant turn and every tool-result write: `latency_ms` (wall-clock per step), `tokens_in / tokens_out` (from the LLM response when available), `model_name`, `prompt_version` (read from active config when Stream B lands; else `"default"`), and `retrieval_chunk_ids` (the chunk PKs returned by the last `search_knowledge_base` call in this turn, if any).

## Acceptance Criteria

- [ ] Every `Message` written by `AgentLoop` carries `latency_ms` and `model_name` (non-null on assistant + tool turns).
- [ ] `tokens_in` / `tokens_out` are non-null when the LLM client returned usage stats; null otherwise (no crash).
- [ ] `retrieval_chunk_ids` reflects the chunks returned by RAG on that turn.
- [ ] Existing tests pass; one new test asserts that a real agent run populates all five fields.

## Technical Details

- New helper: `_finalize_message(...)` that takes raw response + timing and writes the Message row.
- Use `time.perf_counter()` around the LLM call and around each tool call.
- Read `prompt_version` from `agent/prompt_config.get_active_config()` if available; else `"default"`.
