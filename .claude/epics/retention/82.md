# Task: Admin endpoints — customers list / detail / search / add-note

## Description

Staff-gated DRF endpoints powering the admin UI:
- `GET /api/admin/customers/?q=&page=` — paginated list, search across name (icontains) and phone (normalised exact-match).
- `GET /api/admin/customers/{id}/` — detail with embedded recent visits (last 50) and notes.
- `POST /api/admin/customers/{id}/notes/` — add a note (body); response is the new note.

## Acceptance Criteria

- [ ] List endpoint paginates at 50 rows/page, search runs case-insensitive on name and tries `normalize_phone(q)` against `phone` for exact lookup.
- [ ] Detail endpoint embeds visits + notes in one response (no N+1 on the frontend).
- [ ] Add-note endpoint records `author` from the authenticated staff user.
- [ ] All three endpoints reject non-staff with 403.
- [ ] 6+ tests: list, search by name, search by phone, detail shape, add-note auth, non-staff 403.

## Technical Details

- `backend/api/admin_customers.py`.
- Use `select_related('customer')` + `prefetch_related('notes')` to keep the detail endpoint a single round trip.
