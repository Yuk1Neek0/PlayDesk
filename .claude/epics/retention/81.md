# Task: Booking signal — maintain Customer counters

## Description

`post_save` signal on `Booking` keeps `Customer.total_visits` and `Customer.last_visit_at` in sync. Cancelled bookings DON'T count toward the visit total; status transitions are handled correctly.

## Acceptance Criteria

- [ ] Creating a confirmed booking increments the customer's `total_visits` and updates `last_visit_at`.
- [ ] Cancelling a confirmed booking decrements `total_visits`.
- [ ] Re-confirming a previously-cancelled booking increments again (no over-counting).
- [ ] Bookings with `customer_id IS NULL` (legacy path before backfill) are skipped silently.
- [ ] 4+ tests covering create, cancel, re-confirm, null-customer cases.

## Technical Details

- Signal lives in `backend/core/signals.py`. Connected in `core/apps.py::ready()`.
- Use a status-transition diff (compare `pre_save` vs `post_save`) to avoid double-counting on updates that don't change status.
