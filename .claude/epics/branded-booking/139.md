# Task: fetchStoreBrand() SSR helper + types + tests

## Description

The shared frontend helper that both the booking page and the QR page will use. SSR-side fetch (server component) against the new public endpoint. Includes the type, error handling, and the cache directive matching the endpoint's `max-age=60`.

## Acceptance Criteria
- [ ] `frontend/src/lib/store-brand.ts` exports:
   - `interface StoreBrand { name: string; logo_url: string | null; accent: string | null; }`
   - `fetchStoreBrand(): Promise<StoreBrand>`
- [ ] Implementation uses `BACKEND_ORIGIN` for the URL (matches the v3 SSR-fetch fix on `/qr/[slug]`).
- [ ] Cache directive: `cache: "default"` (matches the endpoint's 60s cache; do NOT use `no-store`).
- [ ] On fetch failure / non-2xx: returns `{ name: "PlayDesk", logo_url: null, accent: null }` — graceful degrade to the default.
- [ ] Tests in `frontend/src/lib/store-brand.test.ts`:
   - Mocked 200 with valid payload → returns the payload as-is.
   - Mocked 200 with `accent: null` → returns `accent: null`.
   - Mocked 500 → returns the default-degraded shape (no thrown exception).
   - Mocked network error → returns the default-degraded shape.

## Technical Details

- The graceful-degrade default is important: a transient backend hiccup must not break the booking page render. The page renders with default branding instead.
- The cache directive matches the endpoint's `max-age=60` to maximise Next.js fetch dedup.
- The type lives in `lib/store-brand.ts` rather than `types/api.d.ts` because this isn't part of the generated OpenAPI surface — it's a small UI-facing helper.
