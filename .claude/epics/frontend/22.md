# Task: Admin dashboard

## Description

Wire the `/admin` shell into a staff dashboard: a live conversations panel and a
bookings table sorted newest-first. New bookings appear without a manual
refresh (polling or refetch).

## Acceptance Criteria
- [ ] Bookings table lists bookings sorted by `created_at` descending.
- [ ] Live conversations panel shows active chat sessions.
- [ ] A new booking appears without a manual page refresh.
- [ ] Component tests pass against mocked API responses.
- [ ] CI green: lint, typecheck, vitest, build.

## Technical Details

- Use the generated client from task 001 for the admin endpoints.
- Auto-update via polling or refetch — keep it simple, no websockets.
- Tailwind defaults only.
- Touches only the `/admin` route files — conflict-free with tasks 002 and 003.

## Dependencies

- Task 001 (shared API client).
