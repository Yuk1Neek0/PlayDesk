# Task: API client & SSE hook

## Description

Build the shared frontend data layer: TypeScript types and a typed fetch client
generated from `docs/contracts/openapi.yaml`, plus a typed SSE reader hook that
parses the streaming protocol in `docs/contracts/sse-protocol.md`. Every page
task (002–004) builds on this; it lands first so the three pages can then run
in parallel.

## Acceptance Criteria
- [ ] TypeScript request/response types generated from `docs/contracts/openapi.yaml` (no hand-written types).
- [ ] Typed fetch wrappers cover availability, booking, resources, and admin endpoints.
- [ ] An SSE client hook reads the protocol's event types and exposes streaming tokens + in-flight tool-call hints to React.
- [ ] Unit tests run against mocked `fetch` and a mocked SSE stream — no live backend needed.
- [ ] CI green: `npm run lint && npm run typecheck && npm test && npm run build`.

## Technical Details

- Use an OpenAPI type generator wired into a `package.json` script so types regenerate from the frozen contract.
- The SSE hook should be non-blocking: tokens render incrementally and tool-call hints surface without freezing the UI.
- Keep wrappers thin — error surface and base-URL config only.

## Dependencies

- Frozen `foundation` contracts: `docs/contracts/openapi.yaml`, `docs/contracts/sse-protocol.md`.
