# Task: twilio_voice_webhook view + bilingual TwiML + Conversation creation

## Description

The Twilio Voice webhook that answers inbound calls. Returns a static bilingual `<Say>` greeting via TwiML and records a `Conversation` row with `channel='phone'`. Links to a `Customer` when the caller's normalised phone matches one. No STT, no TTS, no agent turn.

## Acceptance Criteria
- [ ] `backend/api/webhooks_twilio.py` exports `twilio_voice_webhook(request)` view.
- [ ] `POST /api/webhooks/twilio/voice/` registered in `backend/api/urls.py`.
- [ ] Verifies signature via `verify_twilio_signature` from the shared helper.
- [ ] Parses Twilio's POST body for `From` (E.164 phone) and `CallSid`.
- [ ] Normalises `From` via `core.phone.normalize_phone`. Looks up `Customer` by `(store=default, phone=normalized)`. Links if found; otherwise customer=None.
- [ ] Creates `Conversation(channel='phone', customer=..., customer_identifier=normalized_phone, status='completed')`. Stores `CallSid` in the conversation's metadata field if one exists, otherwise skip (don't add a new field for v5).
- [ ] Returns TwiML:
   ```xml
   <Response>
     <Say voice="Polly.Joanna" language="en-US">You've reached PlayDesk. Voice is coming soon — please text us at this number for now.</Say>
     <Say voice="Polly.Zhiyu" language="cmn-CN">中文您也可以發短信給我們。</Say>
   </Response>
   ```
- [ ] Returns `503 not_configured` when `TWILIO_AUTH_TOKEN` is missing (same posture as SMS/WhatsApp).
- [ ] Returns 403 on tampered signature.
- [ ] Tests in `tests/test_voice_webhook.py`:
   - Signed valid POST from a known customer's phone → 200 + TwiML, Conversation row linked to that customer.
   - Signed valid POST from an unknown phone → 200 + TwiML, Conversation row with customer=None.
   - Tampered signature → 403, no Conversation row written.
   - Missing `TWILIO_AUTH_TOKEN` → 503, body `{"error":"not_configured"}`.
   - TwiML body contains both English and Mandarin `<Say>` elements with the right `language` attrs.

## Technical Details

- The view is intentionally side-effect-y: it creates a Conversation row even on a one-shot call. Justification: the admin filter for `channel=phone` needs data to filter, and call-attempt visibility is itself useful staff data.
- The `<Say>` voice + language pair (`Polly.Joanna` + `en-US`, `Polly.Zhiyu` + `cmn-CN`) is a Twilio-supported combination; documented in the TwiML body so the message can be edited without other code changes.
- Do NOT call `AgentLoop.run` from this view — voice doesn't go through the agent in v5. That's the whole point of "scaffold."
