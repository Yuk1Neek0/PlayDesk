# Task: REST API — resources, availability, booking CRUD, admin

## Description

Implement the DRF REST API in a new `backend/api/` app, matching `docs/contracts/openapi.yaml`.

## Acceptance Criteria
- [ ] `GET /api/resources/` — list resources, filterable by `type`.
- [ ] `GET /api/resources/{id}/availability/?date=YYYY-MM-DD` — open slots = business hours − existing bookings.
- [ ] `POST /api/bookings/` — create booking; returns `409` when the DB `EXCLUDE` constraint rejects an overlap (catch `IntegrityError`, do not pre-check in Python).
- [ ] `GET /api/bookings/`, `GET/PATCH/DELETE /api/bookings/{id}/`.
- [ ] `GET /api/admin/conversations/`, `GET /api/admin/bookings/` (newest first).
- [ ] Responses match the OpenAPI schemas, including `Booking.status` with `pending_payment`.
- [ ] pytest covers the happy paths and the concurrent-overlap `409`.

## Technical Details
- All code under `backend/api/` with its own `urls.py`; do NOT edit `config/urls.py` or `config/settings.py` — report the include + `INSTALLED_APPS` entry needed.
- Availability computes from `Store.business_hours` minus `Booking` rows for the date.
