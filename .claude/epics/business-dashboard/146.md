# Task: <MetricCard /> reusable component + vitest test

## Description

Small reusable React component for the dashboard's six cards. Each card: big number + label + optional trend chip + optional secondary line + click-target deep-link wrapper.

## Acceptance Criteria
- [ ] `frontend/src/components/admin/metric-card.tsx` exports:
   - `interface MetricCardProps { label: string; value: string | number; href?: string; trendPct?: number | null; secondary?: string; }`
   - `MetricCard(props): JSX.Element`.
- [ ] When `href` is set, the card wraps in a Next `<Link>`.
- [ ] When `trendPct` is set and non-null, a small chip renders: `+X%` (green-ish) for positive, `-X%` (red-ish) for negative, `±0%` for zero, hidden for null.
- [ ] When `secondary` is set, renders below the value in smaller text.
- [ ] Uses existing `pd-stat` / `pd-card` design tokens — no new CSS file.
- [ ] vitest test in `src/components/admin/metric-card.test.tsx`:
   - Renders the label + value.
   - With `href`, wraps in a link.
   - With `trendPct=15.5`, renders a `+15.5%` chip with the up-trend style.
   - With `trendPct=-5`, renders `-5.0%` with the down-trend style.
   - With `trendPct=null`, no chip renders.

## Technical Details

- This component is intentionally small + presentational — no data fetching, no polling. The composite strip (task 005) handles fetching.
- Reuse existing CSS classes from `pd-ui.css` (or wherever the design tokens live). No new tokens.
- Accessibility: card should have an `aria-label` combining label + value so screen readers don't have to splice the big-number presentation.

## Dependencies

- Task 003 (typed client) — needed for the strip in 005, but `MetricCard` itself is independent and can be written in parallel.
