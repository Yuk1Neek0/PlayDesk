# Task: <BusinessDashboardStrip /> composite + 60s polling + mount on /admin

## Description

The dashboard strip itself: fetches the composite metrics, renders six `<MetricCard />`s in a responsive grid, polls every 60s, shows a skeleton on first load. Mounts above the existing "Tonight at PlayDesk" section on `/admin`.

## Acceptance Criteria
- [ ] `frontend/src/components/admin/business-dashboard-strip.tsx` exports `BusinessDashboardStrip()`.
- [ ] On mount: calls `adminGetBusinessMetrics()`, shows a skeleton (six placeholder cards) until the first response arrives.
- [ ] After first load: renders six cards in a `pd-grid` (or equivalent), wrapping responsively on narrow viewports.
- [ ] Polls every 60s using `setInterval`; cleans up on unmount.
- [ ] If a poll is in flight when the next interval fires, the previous one is cancelled (use AbortController).
- [ ] Each card's `href` deep-links per the PRD: bookings_today → `/admin/bookings?date=today`, revenue → `/admin/bookings?status=completed`, customers → `/admin/customers`, outbound → no link in v5 (admin/outbound list is per-customer only), QR → `/admin/qr`.
- [ ] `frontend/src/app/admin/page.tsx` mounts `<BusinessDashboardStrip />` ABOVE the existing "Tonight at PlayDesk" section.
- [ ] Existing `/admin` layout is preserved — "Tonight at PlayDesk", "All bookings" sections still render in their current positions.
- [ ] `npm run lint && npm run typecheck && npm run build` clean.
- [ ] vitest test in `src/components/admin/business-dashboard-strip.test.tsx`:
   - Renders six skeleton cards before the fetch resolves.
   - Renders six populated cards after the mocked fetch resolves.
   - Polling: advancing the mock timer by 60s triggers a second fetch.
- [ ] Existing `frontend/e2e/admin.e2e.ts` Playwright tests pass unchanged (the strip is additive, doesn't move the existing "All bookings" heading).

## Technical Details

- Polling cadence: 60s matches the PRD. Use `useEffect` with `setInterval` + cleanup; AbortController on the in-flight fetch.
- Skeleton style: six `<div class="pd-stat pd-skeleton">` placeholders — same shape as the populated cards.
- Card label/value formatting:
   - `bookings_today`: value = `count`, secondary = `+/- X% vs yesterday` when trend present.
   - `revenue_window`: value = formatted CAD (e.g. `CA$ 12,450`), secondary = `last N days`.
   - `new_customers_window`: value = `count`, secondary = `last N days`.
   - `outbound_24h`: value = `sent`, secondary = `N failed / N queued`.
   - `qr_window`: value = `engagement_pct` formatted as `XX.X%`, secondary = `N scans / M clicks · last 7 days`.
