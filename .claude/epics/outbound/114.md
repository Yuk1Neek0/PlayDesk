# Task: OutboundChannelAdapter ABC + TwilioSmsOutboundAdapter + adapter registry

## Description

The outbound counterpart to the v3 inbound `ChannelAdapter`. One ABC, one Twilio implementation, one registry â€” symmetric to inbound but not coupled (their interfaces are different shapes).

This task is independent of task 001 (no DB writes); can develop in parallel.

## Acceptance Criteria
- [ ] `backend/agent/channels/outbound_base.py` defines `OutboundChannelAdapter(ABC)` with a `channel: ClassVar[str]` and an abstract `send(to_identifier: str, body: str, metadata: Mapping[str, Any] | None = None) -> OutboundSendResult`.
- [ ] `OutboundSendResult` dataclass with `ok: bool`, `provider_message_id: str | None`, `reason: str | None`.
- [ ] Pure Python â€” no Django imports (mirrors the inbound `ChannelAdapter` design from `backend/agent/channels/base.py`).
- [ ] `backend/agent/channels/twilio_sms_outbound.py` provides `TwilioSmsOutboundAdapter(OutboundChannelAdapter)`. Uses the same `TWILIO_AUTH_TOKEN` / `TWILIO_ACCOUNT_SID` / `TWILIO_FROM_NUMBER` env vars as the inbound adapter.
- [ ] When any of those env vars are missing, `send()` returns `OutboundSendResult(ok=False, provider_message_id=None, reason="not_configured")` â€” never raises.
- [ ] On a successful Twilio send, `provider_message_id` is the Twilio SID.
- [ ] Adapter registry in `backend/agent/channels/__init__.py` (or a sibling `registry.py`) â€” `get_outbound_adapter(channel: str) -> OutboundChannelAdapter`. Returns the registered adapter or raises `KeyError` for unknown channels.
- [ ] A `LoggingOutboundAdapter` fake for tests â€” registered under a `test` channel, records every send to an in-memory list for assertion.
- [ ] Tests cover: ABC enforcement (`send()` is abstract); Twilio `not_configured` path returns the right shape; Twilio happy path returns the SID; registry lookup; registry KeyError for unknown channel.

## Technical Details

- File layout mirrors inbound: `backend/agent/channels/{outbound_base,twilio_sms_outbound}.py`.
- The Twilio adapter calls `twilio.rest.Client(account_sid, auth_token).messages.create(to=to_identifier, from_=from_number, body=body)`. Wrapped in `try / except twilio.base.exceptions.TwilioRestException as e: return SendResult(ok=False, reason=str(e))`.
- `LoggingOutboundAdapter` is in `backend/agent/channels/_test_helpers.py` so it doesn't ship in prod imports.
- Do not introduce a Celery dependency.
