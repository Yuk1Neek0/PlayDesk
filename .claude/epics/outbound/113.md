# Task: Migration — OutboundMessage model + Store.quiet_hours_start/end fields

## Description

Create a new `outbound` Django app and ship its initial migration: the `OutboundMessage` queue table plus two new fields on `Store` for quiet hours. Additive only, no backfill.

## Acceptance Criteria
- [ ] New Django app: `backend/outbound/` with `apps.py`, `__init__.py`, `models.py`, `migrations/`. Registered in `INSTALLED_APPS`.
- [ ] `OutboundMessage` fields: `customer (FK CASCADE)`, `channel (CharField, choices: sms|web_chat, default sms)`, `template_key (CharField max 64)`, `body (text)`, `status (CharField, choices: queued|sent|failed|cancelled, default queued)`, `scheduled_for (datetime, default timezone.now)`, `sent_at (datetime, nullable)`, `failure_reason (text, blank)`, `reference (CharField max 100, blank)`, `created_at (auto_now_add)`.
- [ ] Composite index on `(status, scheduled_for)` so the sender query is one range scan.
- [ ] Index on `(customer, -created_at)` for the customer-detail list query.
- [ ] `Store.quiet_hours_start` (TimeField, default `time(22, 0)`).
- [ ] `Store.quiet_hours_end` (TimeField, default `time(8, 0)`).
- [ ] Migration applies clean forward + reverse.
- [ ] `verbose_name` on `id` fields.
- [ ] Existing 157-test backend suite passes.

## Technical Details

- App location: `backend/outbound/`.
- `OutboundMessage.status` uses `models.TextChoices` for type safety.
- The `(status, scheduled_for)` composite index is critical — the sender's hot query is `WHERE status='queued' AND scheduled_for <= now() ORDER BY scheduled_for LIMIT 200`.
- The quiet-hours fields live on `Store` (not `OutboundMessage`) so a store-wide policy change applies to all future sends without rewriting queued rows.
