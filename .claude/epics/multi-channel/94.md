# Task: TwilioSmsAdapter + signed webhook + 503 fallback

## Description

Second concrete adapter plus `POST /api/webhooks/twilio/sms/`. Verifies the Twilio signature using `TWILIO_AUTH_TOKEN`. When the token is unset, the endpoint returns `503 not_configured` cleanly so CI without secrets stays green.

## Acceptance Criteria

- [ ] Valid signed payload → 200 + TwiML reply containing the assistant's response.
- [ ] Tampered signature → 403 with `{"error":"invalid_signature"}`.
- [ ] `TWILIO_AUTH_TOKEN` unset → 503 `{"error":"not_configured"}`, no traceback in logs.
- [ ] Conversation row stored with `channel='sms'` and the inbound `From` phone as `customer_identifier` (normalised via `normalize_phone()` if `retention` is in main; else internal helper).
- [ ] 3 tests: happy / tamper / not-configured.

## Technical Details

- `backend/agent/channels/twilio_sms.py` + `backend/api/webhooks/twilio_sms.py`.
- Signature verification via `twilio.request_validator.RequestValidator`.
