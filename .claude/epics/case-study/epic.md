## Overview

The interview-ready packaging tail of v2. Lands only after A, B, C, D merge — its architecture diagram and screenshots must reflect shipped state, not aspirations. Output: a polished `README.md`, an `architecture.md` with Mermaid diagrams, an `ai-design-decisions.md`, a `demo-script.md` with 3-minute and 10-minute cuts, and a `screenshots/` directory.

## Architecture Decisions

- **No code changes.** Pure documentation + assets. Zero risk to the running system.
- **Mermaid only.** GitHub renders Mermaid natively — no plugin, no image hosting.
- **Two-tier demo script.** 3-minute for interviews; 10-minute for technical deep-dives. Same beats, different depth.
- **README is the reviewer surface; CLAUDE.md stays as the agent surface.** Two different audiences, two different docs.
