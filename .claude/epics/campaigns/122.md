# Task: Thin send interface — real (outbound-delegating) + stub + import-time switch

## Description

The single abstraction that lets `campaigns` ship in parallel with `outbound`. One Python function, two implementations, picked at import time. If `outbound.api.enqueue_message` is importable, the real impl delegates to it; otherwise a logging stub returns success and the audit trail still records the attempt.

This task mirrors the v3 retention↔multi-channel pattern where the SMS adapter shipped with its own `normalize_phone` stub until retention landed.

## Acceptance Criteria
- [ ] `backend/campaigns/send.py::send_campaign_message(customer, body, reference) -> SendResult` returns a dataclass with `ok: bool, provider_message_id: str | None, reason: str | None`.
- [ ] Implementation picked at module import time:
  - Try `from outbound.api import enqueue_message`. On success, bind the real impl: calls `enqueue_message(customer, template_key="campaign", context={"body": body}, channel="sms", reference=reference)`, returns `{ ok: True, provider_message_id: str(msg.id), reason: None }`.
  - On `ImportError`, bind the stub: logs `[campaigns] stub send (outbound not installed): <reference>` once per import + per send, returns `{ ok: True, provider_message_id: None, reason: None }`.
- [ ] A `_get_impl()` helper exposes the selected impl for tests to swap (lets tests force either impl regardless of installed apps).
- [ ] The `outbound` template registry includes a `campaign` entry whose template is the literal `"{body}"` (set by outbound's task 003). When outbound is present, this task assumes that — no work to do here.
- [ ] Tests cover both paths:
  - With a monkeypatched-importable `outbound`: real path is selected; `send_campaign_message` writes to the outbound queue with `template_key="campaign"`.
  - With a monkeypatched-missing `outbound`: stub path is selected; the log line is captured; no DB writes happen.
- [ ] Test fixture: a context manager `force_send_impl(impl)` that temporarily binds the selected impl and restores the original on exit, so test ordering doesn't matter.

## Technical Details

- File: `backend/campaigns/send.py`.
- The import-time switch is implemented with a module-level `try: ... except ImportError: ...` that sets a module-private `_impl` variable.
- The test fixture is needed because tests for the runner (task 004) need both branches exercised in the same test run.
- Do not introduce any other coupling to `outbound` — no imports of `outbound.models`, no reaching into outbound internals.

## Dependencies

- Task 001 (so the `Campaign` / `CampaignRun` types the dataclass references exist).
- Soft: outbound's task 003 (`enqueue_message`). If outbound has not landed when this ships, the stub path covers it.
