# Task: Migration — Segment + Campaign + CampaignRun

## Description

Create the `campaigns` Django app and ship its initial schema. Additive only.

## Acceptance Criteria
- [ ] New Django app: `backend/campaigns/` with `apps.py`, `models.py`, `migrations/`. Registered in `INSTALLED_APPS`.
- [ ] `Segment` fields: `store (FK CASCADE)`, `name (CharField max 200)`, `filter (JSONField, default dict)`, `created_by (FK AUTH_USER_MODEL, SET_NULL)`, `created_at (auto_now_add)`.
- [ ] `Campaign` fields: `store (FK CASCADE)`, `name (CharField max 200)`, `segment (FK PROTECT)`, `body_template (text)`, `scheduled_for (datetime, default now)`, `status (CharField, choices: draft|scheduled|sending|sent|cancelled, default draft)`, `created_by (FK AUTH_USER_MODEL, SET_NULL)`, `sent_by (FK AUTH_USER_MODEL, SET_NULL, nullable)`, `sent_at (datetime, nullable)`, `recipient_snapshot_count (PositiveIntegerField, default 0)`.
- [ ] `CampaignRun` fields: `campaign (FK CASCADE)`, `customer (FK CASCADE)`, `status (CharField, choices: queued|sent|failed|skipped_optout, default queued)`, `outbound_message_id (CharField max 100, blank)`, `failure_reason (text, blank)`, `created_at (auto_now_add)`, `sent_at (datetime, nullable)`.
- [ ] Unique constraint on `(campaign, customer)` — prevents double-send per recipient.
- [ ] Index on `(campaign, status)` for the per-recipient status table.
- [ ] Migration applies clean forward + reverse.
- [ ] `verbose_name` on `id` fields.
- [ ] Existing backend test suite passes.

## Technical Details

- App location: `backend/campaigns/`.
- The `segment` FK uses `on_delete=PROTECT` so a saved segment cannot be deleted while any campaign references it — deleting a segment requires either deleting its campaigns first or unlinking them.
- `sent_by` is separate from `created_by` so the audit log captures both authorship and the "who pulled the trigger" identities (often different in v4 workflows).
- `recipient_snapshot_count` is written at send time, not at draft time — see task 004.
