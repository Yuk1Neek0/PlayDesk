# Task: Quality report generator — docs/reports/ai-quality-report.md

## Description

CLI entry point that runs the existing eval set, joins with the last N days of trace metadata, and emits a deterministic `docs/reports/ai-quality-report.md`.

## Acceptance Criteria

- [ ] `python -m evals.quality_report` (or equivalent `manage.py` script) produces `docs/reports/ai-quality-report.md`.
- [ ] Output sections: per-prompt-version table, per-model table, latency distribution, cost summary, a "what changed and why" footer.
- [ ] Running twice produces an identical file (idempotent — no timestamps in the body).
- [ ] When only one prompt version exists, the per-version table degrades to a single row labelled clearly.

## Technical Details

- `backend/evals/quality_report.py`.
- Aggregation through `evals.aggregates`.
- Markdown rendered through f-strings — no Jinja dep.
