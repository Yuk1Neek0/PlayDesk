# Task: Environment smoke check

## Description

Confirm the full PlayDesk stack boots and is in a verifiable state before the four verification streams run. This is the blocking prerequisite for the rest of the epic: if the stack does not come up, every downstream stream is wasted effort.

## Acceptance Criteria
- [ ] `docker compose up` brings `db`, `backend`, and `frontend` to a healthy state.
- [ ] Django migrations apply cleanly; `btree_gist` extension and the `EXCLUDE USING gist` constraint on `Booking` exist.
- [ ] The knowledge base is ingested (`ingest_kb`) and `KnowledgeChunk` rows exist with embeddings, in both `lang` values.
- [ ] Seed data exists: at least one `Store`, multiple `Resource`s across types, `GameMenu` entries.
- [ ] `backend` reachable on its port; `frontend` serves `/`, `/chat`, `/admin` without a 500.
- [ ] Required env/secrets are present: LLM API key, Stripe test-mode keys — documented if missing.
- [ ] A short environment record is written to `updates/001/` listing service health, row counts, and any gaps.

## Technical Details

- Run from repo root: `docker compose up`. Inspect logs for migration and ingest output.
- If seed data is absent, identify the seeding mechanism (fixture / management command) and run it; record what was needed.
- pgvector index presence (HNSW or IVFFlat fallback) should be confirmed on `KnowledgeChunk`.
- Any fix needed to make the stack boot (compose config, missing migration, env wiring) is made here, in-stream.

## Dependencies

- All four build epics merged to `main` — satisfied.

## Effort Estimate
- Size: S
- Hours: 2

## Definition of Done
- [ ] Stack boots reproducibly
- [ ] Environment record written to `updates/001/`
- [ ] Any boot-blocking fixes committed
- [ ] Downstream streams unblocked
