# Task: Integration & sign-off

## Description

Reconcile the four streams' in-stream fixes on the epic branch, confirm they coexist, and produce the consolidated verification record that signs off the demo as safe to present.

## Acceptance Criteria
- [ ] Any shared-file fixes flagged by Streams A–D (`config/settings.py`, `config/urls.py`, `frontend/src/types/api.d.ts`) are reconciled into a single coherent state.
- [ ] Backend `pytest` passes in full (all apps) against the merged branch.
- [ ] Frontend `npm run lint && npm run typecheck && npm test && npm run build` all pass against the merged branch.
- [ ] `ruff check` + `ruff format` clean; ESLint clean.
- [ ] The full end-to-end demo path runs green on a fresh `docker compose up`: book via `/chat` → appears in `/admin` with no refresh.
- [ ] A consolidated verification record is written: every dev plan criterion (§1.1–1.5, §2.1–2.4) → pass, with a pointer to its per-stream evidence.
- [ ] Epic frontmatter and `Tasks Created` checklist updated; epic ready to merge to `main`.

## Technical Details

- Pull each stream's branch/commits onto the epic branch; resolve conflicts in shared files deliberately, not mechanically.
- Re-run both full suites on the reconciled tree — a fix that passed in isolation may regress once merged.
- Assemble `updates/006/verification-record.md` from the four per-stream records into one demo checklist.
