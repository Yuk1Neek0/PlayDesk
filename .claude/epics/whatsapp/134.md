# Task: TwilioWhatsAppOutboundAdapter + registry + tests

## Description

The outbound counterpart to `TwilioSmsOutboundAdapter`. Sends WhatsApp messages via the Twilio SDK, prepending `whatsapp:` to both `From` and `To`. Uses a new `TWILIO_WHATSAPP_FROM` env var. Returns `not_configured` cleanly when missing.

## Acceptance Criteria
- [ ] `backend/agent/channels/twilio_whatsapp_outbound.py` defines `TwilioWhatsAppOutboundAdapter(OutboundChannelAdapter)` with `channel = "whatsapp"`.
- [ ] Reads `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`, and the new `TWILIO_WHATSAPP_FROM` env vars.
- [ ] When `TWILIO_WHATSAPP_FROM` (or either of the SID/token) is missing, `send()` returns `OutboundSendResult(ok=False, provider_message_id=None, reason="not_configured")` — never raises.
- [ ] On a successful Twilio send, returns the message SID as `provider_message_id`.
- [ ] Twilio API call uses `whatsapp:<TWILIO_WHATSAPP_FROM>` for `from_` and `whatsapp:<to_identifier>` for `to`.
- [ ] Outbound adapter registry (`backend/agent/channels/registry.py`) gains an outbound entry: `'whatsapp' → TwilioWhatsAppOutboundAdapter`.
- [ ] Tests in `tests/test_whatsapp_outbound_adapter.py`:
   - Missing env var → `not_configured` result, no API call attempted.
   - Mocked Twilio success → ok=True with the right SID.
   - Mocked Twilio failure → ok=False with the failure reason string.
   - The Twilio API call uses the `whatsapp:` prefix on both `from_` and `to`.
- [ ] `TWILIO_WHATSAPP_FROM` documented in `docs/outbound-cron.md` (or wherever the SMS env vars are documented).

## Technical Details

- Reuse the same Twilio Python client setup as `twilio_sms_outbound.py`; the only API difference is the `whatsapp:` prefix on both sides.
- Twilio sandbox: `TWILIO_WHATSAPP_FROM=+14155238886` is the documented sandbox sender.
- Outside the customer-initiated 24-hour session window, Twilio requires pre-approved templates. v5 free-form messaging works inside the window — out-of-window template support is explicitly out of scope.
