# Task: Backend scaffold & dev environment

## Description

Create the Django + DRF backend skeleton and the Dockerized development environment. This is the serial spine of the foundation epic — task 002 and 005 build on top of it.

## Acceptance Criteria
- [ ] `backend/` contains a Django 4.x project (`config/` settings package, `manage.py`) with DRF installed.
- [ ] `requirements.txt` (django, djangorestframework, psycopg, pgvector, pydantic, django-environ) and `requirements-dev.txt` (ruff, pytest, pytest-django, pytest-cov).
- [ ] Root `docker-compose.yml` defines `db` (`pgvector/pgvector:pg16`), `backend`, and `frontend` services; `.env.example` documents every variable.
- [ ] `ruff` config and `pytest`/`pytest-django` config present; `pytest` runs (zero tests is acceptable) and `ruff check .` passes.
- [ ] `python manage.py migrate` succeeds against the Compose DB; settings read `DATABASE_URL`.
- [ ] DB init enables the `vector` and `btree_gist` extensions.

## Technical Details
- Settings package `config/`, split-friendly; `DATABASE_URL` via `django-environ`.
- Apps namespace: a `playdesk` app (or `core`) to host models in task 002.
- Compose: `backend` depends_on `db` healthcheck; `frontend` builds from `frontend/`.
- Keep `.env.example` the single source of dev config truth.

## Dependencies
None — first task in the epic.
