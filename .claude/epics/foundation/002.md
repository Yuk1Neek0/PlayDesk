# Task: Data layer: models, migrations & overlap constraint

## Description

Implement the seven PlayDesk models, their migrations, the database-enforced booking-overlap constraint, and seed fixtures. This freezes the **data-schema contract** for all of Wave 1.

## Acceptance Criteria
- [ ] Models implemented per the dev-plan ER diagram: `Store`, `Resource`, `GameMenu`, `Booking`, `Conversation`, `Message`, `KnowledgeChunk`.
- [ ] `Booking` has an `EXCLUDE USING gist` constraint over `(resource_id WITH =, tstzrange(start_time, end_time) WITH &&)` (requires `btree_gist`).
- [ ] `KnowledgeChunk` has a pgvector `embedding` column plus `category`, `source`, `lang` fields.
- [ ] Forward-compatible enums: `Booking.status` includes `pending`, `confirmed`, `cancelled`, `pending_payment`; `Booking.source` ∈ `manual|agent`.
- [ ] `Message` stores `role`, `content`, and a JSONB `tool_call_data` field.
- [ ] Migrations apply cleanly; seed fixtures load stores, resources, and game-menu rows.
- [ ] A test proves concurrent inserts at the same `(resource_id, time)` produce exactly one success and one `IntegrityError`.

## Technical Details
- Create the `btree_gist` extension in a migration before the constraint migration.
- Use `django.contrib.postgres.constraints.ExclusionConstraint` with `RangeOperators`.
- Vector column via the `pgvector` Django integration; dimension 1536 (`text-embedding-3-small`).
- Seed data via a fixture file or an idempotent management command.
